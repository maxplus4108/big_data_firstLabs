
## Выполненные задачи
* Создан файл `.env` для защиты паролей (добавлен в `.gitignore`).
* В `docker-compose.yml` значения заменены на переменные `${VAR}`.
* Настроен проброс портов и именованный том через переменные.
* В контейнер передан MD5-хэш имени (`SECRET_HASH`).

## Команды
* **Запуск:** `docker-compose up --build -d`
* **Вход в терминал контейнера:** `docker exec -it postgres_logs_db bash`
* **Просмотр переменных внутри:** `printenv`
* **Вывод отфильтрованных переменных окружения:** `printenv | grep -E "POSTGRES|SECRET_HASH"`

## Ответы на вопросы задания

**1. В каких случаях потребуется проброс портов?**
Для доступа к БД вне Docker. Это нужно для подключения через визуальные клиенты (DBeaver, pgAdmin), выполнения внешних скриптов.

**2. Каких переменных нет в выводе `printenv` и почему?**
Внутри контейнера отсутствуют `DB_PORT` и `VOLUME_NAME`. 
Эти переменные считываются утилитой `docker-compose` на  ПК для настройки инфраструктуры (создания диска и открытия портов).
